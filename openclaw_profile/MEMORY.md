## 重要设置
- **子 agent 超时必须设为 3 小时 (10800 秒)**，不要用默认的 10 分钟。用户多次强调过。
- **生成 agent 完成后必须自己审核**：生成任务后调用审核流程，不通过的要修到通过为止。不要分成两个阶段。
- **新建任务必须齐全**：每个任务必须包含完整的 task_description.txt、judger_prompt.txt、reference/（audit + evidence + gold_contract + 来源截图）、input/（实际输入文件）。
- **严格 GUI 审查**：任务必须真正依赖 GUI 操作，不能被 CLI/脚本轻易替代。Docker/VS Code 配置/tshark/pvpython/-SILENT 模式等不合格。
- **难度要求**：每个任务必须是 expert 级别，需要 hours 来完成。太简单的不合格。
- 用户要求：**全部跑完再停**，不要中途停下来问。
- 用户要求：继续 scale up tasks，尽量把表里所有 `No tasks` 领域都高难度覆盖，且保持最高输出标准（完整 input/judger/evidence/audit）。
- 当前优先级：**先把现有 500+ 任务全部治理好，再继续扩充**；目标是产出一批高执行度任务完整包。

## Benchmark 关键原则（Tianyu 多次强调）
- **Gold values 必须从数据源提取**（tutorial/文档/论文的表格、图表、文字），绝不能自己运行软件
- **每个任务必须有 evidence**：来源 URL + 来源页面截图 + gold values 截图
- **evidence 放在任务自己的目录下**
- **图表也能做 judge**：从图表读数 + 设宽容度（±5-10%）
- **任务应该依赖 GUI 操作**，但 CLI 软件只要需要配置+运行+提取结果也可以
- **不能被公式直接秒的才合格**
- **修改参数的任务不合格**（文档里没有对应的 gold values）→ 改回原始参数
